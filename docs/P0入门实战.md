# P0 入门实战：从零跑通第一版

> **本阶段目标**：不追求完美，先让「知识库 → 检索 → DeepSeek 回答」在本地跑起来。  
> **预计时间**：第一次约 2～4 小时（含注册 API、排错）。  
> **完成后**：再分给队友会轻松很多，因为流程你已经走过一遍。

---

## 开始前：P0 到底在做什么？

用一张图记住（后面每一行代码都在这张图里）：

```text
你在终端输入一句话
        ↓
   main.py 收到
        ↓
 retriever.py 在 knowledge/*.md 里「搜」相关场馆（P0 是关键词）
        ↓
 main.py 把搜到的文字 + 你的问题 拼成一大段
        ↓
   llm.py 发给 DeepSeek
        ↓
 终端打印推荐结果
```

**P0 故意还没做**：向量检索、多轮记忆、Gradio 网页——那些是 P1 以后的事。

## 第 0 步：确认项目位置

1. 打开 Cursor，左侧资源管理器应能看到文件夹：  
   `校园智能场馆匹配平台`
2. 记下**项目根目录**完整路径，例如：  
   `d:\大学启动\人工智能基础A\大作业\校园智能场馆匹配平台`
3. 后文所有命令都在**这个文件夹里**执行。

**自检**：根目录下能看到 `main.py`、`llm.py`、`knowledge` 文件夹。

---

## 第 1 步：检查 Python 是否可用

1. 在 Cursor 按 **Ctrl + `** 打开终端（或菜单：终端 → 新建终端）。
2. 输入下面命令后回车：

```powershell
python --version
```

3. 应显示 `Python 3.10` 或更高（如 3.11、3.13）。

**若提示找不到 python**：在 Cursor 右下角点 Python 解释器，选已安装的版本；或安装 Python 3.10+ 并勾选「Add to PATH」。

---

## 第 2 步：创建虚拟环境（推荐，避免装乱全局包）

在终端中**逐条**执行（每条回车等它跑完）：

```powershell
cd "d:\大学启动\人工智能基础A\大作业\校园智能场馆匹配平台"
```

（把你的路径换成自己的根目录。）

```powershell
python -m venv .venv
```

```powershell
# 在 Windows PowerShell 下激活虚拟环境的命令（执行后命令行前会出现 (.venv)）
.\.venv\Scripts\Activate.ps1
```

激活成功后，命令行前面会出现 `(.venv)`。

**若 PowerShell 禁止运行脚本**：先执行一次：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

再重新激活虚拟环境。

---

## 第 3 步：安装依赖

确保终端前有 `(.venv)`，然后：

```powershell
# 安装依赖包：这条命令会根据项目目录下的 requirements.txt 文件，一次性安装本项目运行所需的全部第三方库（如 openai、requests 等），保证后续代码能顺利运行。
pip install -r requirements.txt
```

看到 `Successfully installed openai ...` 一类提示即可。

**自检**：

```powershell
pip show openai
```

能显示版本号即成功。

---

## 第 4 步：注册并获取 DeepSeek API Key

1. 浏览器打开：https://platform.deepseek.com  
2. 注册 / 登录。  
3. 进入「API Keys」页面，**创建新 Key**。  
4. **立刻复制** Key 保存到记事本（只显示一次，丢了要重建）。  
5. 确认账户有余额或可用额度（大作业用量很小，一般几毛钱级别）。

⚠️ **不要把 Key 发到群聊、不要提交到 Git**。本项目已用 `.gitignore` 忽略 `config.py`。

---

## 第 5 步：创建 config.py（本地密钥配置）

在**项目根目录**执行：

```powershell
Copy-Item config.example.py config.py
```

用 Cursor 打开 `config.py`，修改这一行：

```python
API_KEY = "sk-xxxxxxxx"   # 粘贴你的真实 Key
```

其余可先保持默认：

```python
API_BASE = "https://api.deepseek.com"
MODEL_NAME = "deepseek-chat"
TEMPERATURE = 0.3
```

**保存文件**（Ctrl+S）。

**自检**：资源管理器里能看到 `config.py`；`config.py` 不在 Git 提交列表里（有 `.gitignore`）。

---

## 第 6 步：只测 API（不跑完整程序）

仍在项目根目录、虚拟环境已激活：

```powershell
python test_llm.py
```

### 期望结果

- 终端打印「调用成功」和模型的一段中文回复。  

### 若失败，对照排查

| 报错大意 | 处理 |
|----------|------|
| 未找到 config.py | 重做第 5 步 Copy-Item |
| 请填写有效的 API_KEY | 检查 Key 是否粘贴完整、无多余空格 |
| 401 / 认证失败 | Key 错误 or 过期，去平台重建 |
| 网络连接失败 | 检查校园网/VPN/防火墙 |
| 429 | 请求太频繁，等一分钟再试 |

**这一步通过 = DeepSeek 已接入成功**，是整个项目最关键的一步。

---

## 第 7 步：测知识库和检索（不调 API，不花钱）

```powershell
python test_retriever.py
```

### 期望结果

- 显示加载了 2 个文件（示例体育馆、示例游泳馆）。  
- 用「羽毛球」检索时，应至少命中体育馆相关文件。

### 理解这里在干什么

- `knowledge/*.md`：每个文件 ≈ 一个场馆说明书。  
- `retriever.py`：在文件**正文**里找是否包含你输入的词（P0 很笨，但足够学习）。  

**小实验**：把查询改成「游泳」，看是否命中游泳馆。

---

## 第 8 步：跑完整程序 main.py

```powershell
python main.py
```

1. 出现提示：`请描述您的活动需求：`  
2. 输入例如：`我们社团约 50 人想打羽毛球，要室内`  
3. 等待几秒，应打印一段推荐文字。  
4. 输入 `quit` 退出。

### 此时数据流（建议你对照代码看一遍）

| 顺序 | 文件 | 发生了什么 |
|------|------|------------|
| 1 | `main.py` | `input` 读入你的句子 |
| 2 | `main.py` | 调用 `retrieve(query)` |
| 3 | `retriever.py` | 扫描 `knowledge/`，算关键词分数 |
| 4 | `main.py` | `build_user_message` 拼「需求 + 场馆资料」 |
| 5 | `main.py` | 组成 `messages`（system + user） |
| 6 | `llm.py` | `chat(messages)` 请求 DeepSeek |
| 7 | `main.py` | `print` 回答 |

**建议阅读顺序**（每文件 5～10 分钟）：  
`main.py` → `retriever.py` → `llm.py` → `config.py`

## 第 10 步：为本校准备真实资料（可慢慢来）

任务书要求 eventually **≥15 段**资料。P0 可先用 2 个示例跑通，再逐步替换：

1. 在 `knowledge/` 新建 `东区体育馆.md`（名字随意）。  
2. 用 Markdown 写清：**位置、容量、适用活动、设备、开放时间、预约方式**。  
3. 删掉或改名「示例-」开头的文件，避免答辩时仍是假数据。  

每增加一个 `.md`，再跑一次 `python test_retriever.py` 看是否被加载。

## 常见问题

**Q：每次都要激活虚拟环境吗？**  
A：新开终端就要再执行 `.\.venv\Scripts\Activate.ps1`。

**Q：Cursor 里能直接点运行吗？**  
A：可以，但要确保用的是项目 `.venv` 里的 Python（看终端/解释器是否带 `(.venv)`）。

**Q：main 回答很慢？**  
A：正常，要等网络 + 模型生成；调试时问题写短一点。

**Q：检索结果不对？**  
A：P0 关键词很粗糙；P2 会改向量检索。先把流程跑通即可。
