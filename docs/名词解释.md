# 名词解释

与本项目相关的核心术语简述，便于阅读 `实现阶段规划.md` 与实验报告。

---

## RAG（检索增强生成）

**Retrieval-Augmented Generation**：先查资料，再让大模型回答。

通用大模型不了解本校场馆细则，直接问易**编造**（幻觉）。RAG 流程为：

```text
用户问题 → 从知识库检索相关段落 → 与问题一并送入 Prompt → DeepSeek 生成回答
```

| 在本项目中 | 对应模块 |
|------------|----------|
| 知识库 | `knowledge/*.md` |
| 切块/索引 | `indexer.py`、`data/chunks.json` |
| 检索 | `retriever.py` |
| 拼 Prompt | `prompts.py` |
| 生成 | `llm.py`（DeepSeek） |

**要点**：RAG 是一套流程，不是单独模型；任务书要求回答有据、库外不编造。

---

## Embedding（文本向量化）

将文本编码为**数值向量**；语义相近的句子，向量距离更近。

用于**语义检索**：按相似度找段落，而非仅匹配关键词（如「羽球」≈「羽毛球」）。

```text
建库：每段资料 → 向量 → 存入 data/embeddings.npz
查询：用户问题 → 向量 → 与库中向量算相似度 → 取 Top-K 段供 RAG 使用
```

| 角色 | 说明 |
|------|------|
| Embedding | 负责「找哪几段资料」 |
| DeepSeek | 负责「根据资料写推荐」（任务书强制 API） |

检索可设**相似度阈值**：过低则视为无相关资料，配合边界处理防胡编。

---

## Gradio

Python 库，用少量代码搭建**网页聊天界面**，用于演示与答辩（任务书进阶加分；必做仅需命令行）。

```text
浏览器 ↔ app.py（Gradio）↔ conversation / retriever / llm（与 CLI 共用核心逻辑）
```

**要点**：`app.py` 只做 UI；业务逻辑与 `main.py` 共用，避免两套实现。

---

## 三者关系（简图）

```text
Gradio（界面）→ 多轮对话 → RAG 检索（常用 Embedding）→ DeepSeek 生成
                      ↑
                 knowledge 知识库
```

| 概念 | 任务书 | 本项目 |
|------|--------|--------|
| RAG | 必做 | 场馆资料检索 + 推荐 |
| Embedding | 进阶 | 语义检索（替代关键词） |
| Gradio | 进阶 | 网页聊天可视化 |
